⚠️
التوكن يبدو هكذا:

لا تشاركه مع أحد ولا ترفعه إلى GitHub أبداً.
🔐
حفظ التوكن في Git Credential Manager (مرة واحدة):
بعد الإعداد التالي Git سيتذكره تلقائياً.
Terminal — حفظ التوكن

  • فتحت Settings → Developer settingsوصلت إلى صفحة Personal access tokens
  • أنشأت التوكن بصلاحية repoنسخته وحفظته في مكان آمن
  • ضبطت credential.helper في Gitسيتذكر التوكن تلقائياً في المرات القادمة

Clone المستودع على جهازك

نسخ مستودع mostafamr2011/index محلياً للتطوير

📂
Clone = نسخ المستودع بالكامل على جهازك
تحصل على كل الملفات + كل تاريخ التعديلات. بعدها تعمل محلياً بحرية ثم ترفع التغييرات.
Terminal — Clone باستخدام التوكن
# الصيغة العامة: # git clone https://[email protected]/USERNAME/REPO.git git clone https://[email protected]/mostafamr2011/index.git # أو clone عادي ثم سيطلب منك التوكن كـ password git clone https://github.com/mostafamr2011/index.git # انتقل للمجلد cd index # تحقق من الاتصال بالـ remote git remote -v origin https://github.com/mostafamr2011/index.git (fetch) origin https://github.com/mostafamr2011/index.git (push) # اعرض الملفات الموجودة ls -la total 112 drwxr-xr-x index/ -rw-r--r-- index.html -rw-r--r-- README.md
⚠️ لا تضع التوكن في الكود مباشرة استخدمه فقط في أمر clone مرة واحدة — بعدها Git يتذكره. لا ترفعه في أي ملف إلى GitHub.
  • Clone ناجح — المجلد index/ موجود على جهازكgit remote -v يُظهر العنوان الصحيح
  • ملف index.html موجود في المجلدهو الملف الرئيسي لموقعك

Branches — العمل بأمان

القاعدة الذهبية: لا تعدّل main مباشرة

main ●─────────────────────────────────● ← موقعك الحالي (لا تلمسه مباشرة) \ / dev ●──────●──────●──────●──────● ← branch تطوير \ feature/x ●──────● ← ميزة محددة
Terminal — إدارة Branches
# اعرض كل الـ branches git branch -a * main remotes/origin/main # أنشئ branch جديد للتطوير واذهب إليه مباشرة git checkout -b dev Switched to a new branch 'dev' # أو بالأمر الجديد (Git 2.23+) git switch -c dev # تحقق أنك على dev git branch * dev main # للتنقل بين branches git switch main git switch dev # احذف branch بعد الدمج git branch -d feature/old
✅
استراتيجية Branches الموصى بها لموقعك:
main ← الموقع الحالي المنشور على github.io
dev  ← التطوير والتجارب
feature/* ← ميزات محددة (قسم جديد، تصميم، إلخ)

Pull — جلب آخر التحديثات

دائماً اسحب قبل أن تبدأ العمل

⚡
القاعدة الذهبية:
قبل أي عمل جديد → git pull
يمنع تعارض الكود ويحافظ على مستودعك محدثاً.
Terminal — Pull scenarios
# جلب آخر التغييرات من main git switch main git pull origin main Already up to date. # أو (نفس الشيء إذا ضبطت upstream) git pull # جلب التغييرات فقط بدون دمج (أكثر أماناً) git fetch origin git diff main origin/main # اعرض الفرق git merge origin/main # ادمج يدوياً # تحديث الـ dev branch من main git switch dev git pull origin main
🔄
fetch vs pull:
git fetch  ← يُحمّل التغييرات فقط (لا يدمج) — للفحص أولاً
git pull   ← يُحمّل ويدمج تلقائياً = fetch + merge
06

Merge · Pull Request — نشر التغييرات

دمج dev في main لتحديث موقعك على github.io

Merge مباشر (محلي)
مناسب إذا تعمل وحدك على المشروع. أسرع وأبسط.
Pull Request (PR)
عبر GitHub — يمنحك فرصة مراجعة التغييرات قبل الدمج.
Terminal — Merge محلي (الطريقة السريعة)
# 1. اذهب لـ main git switch main # 2. تأكد أنه محدث git pull origin main # 3. ادمج dev في main git merge dev Updating a1b2c3d..e4f5g6h Fast-forward index.html | 48 +++++++++++++++++++++++++++++ 1 file changed, 48 insertions(+) # 4. ارفع main إلى GitHub → يتحدث github.io تلقائياً git push origin main To https://github.com/mostafamr2011/index.git a1b2c3d..e4f5g6h main -> main # 5. احذف dev بعد الدمج (اختياري) git branch -d dev
Pull Request — عبر GitHub UI
① بعد git push origin dev — اذهب إلى github.com/mostafamr2011/index ② ستظهر رسالة صفراء: "dev had recent pushes — Compare & pull request" ③ انقر "Compare & pull request" ④ اكتب وصفاً للتغييرات → "Create pull request" ⑤ راجع الـ diff → "Merge pull request" → "Confirm merge" ⑥ انتظر 1-2 دقيقة → mostafamr2011.github.io/index/ يتحدث 🚀
🔀
حل تعارض Merge Conflict:
إذا عدّلت نفس السطر في branch مختلفين، Git يطلب منك الاختيار يدوياً. افتح الملف — ستجد: <<<<<<< HEAD التعديل من main ======= التعديل من dev >>>>>>> dev احذف ما لا تريده + العلامات → git add . && git commit

Cheat Sheet — مرجع سريع يومي

الأوامر الأكثر استخداماً في مكان واحد

Git Cheat Sheet — mostafamr2011/index
## ── SETUP ───────────────────────────────────────────── git clone https://[email protected]/mostafamr2011/index.git cd index ## ── DAILY START ─────────────────────────────────────── git switch dev # اذهب لـ branch التطوير git pull origin main # جلّب آخر التحديثات ## ── WORK ────────────────────────────────────────────── # عدّل index.html في محررك... git status # شاهد التغييرات git diff # شاهد التفاصيل سطراً سطراً git add . # أضف كل التغييرات git commit -m "feat: وصف التعديل" # احفظ git push # ارفع ## ── PUBLISH (dev → main → github.io) ───────────────── git switch main git pull origin main git merge dev git push origin main # ← github.io يتحدث خلال 2 دقيقة ## ── UNDO ────────────────────────────────────────────── git restore index.html # تراجع عن تعديلات غير محفوظة git reset HEAD~1 # تراجع عن آخر commit (يحتفظ بالتعديلات) git reset --hard HEAD~1 # ⚠️ حذف آخر commit والتعديلات ## ── INFO ────────────────────────────────────────────── git log --oneline # سجل التعديلات مختصر git log --oneline --graph # رسم بياني للـ branches git branch -a # كل الـ branches git remote -v # عنوان المستودع البعيد git stash # احفظ تعديلات مؤقتاً بدون commit git stash pop # استعد التعديلات المحفوظة
